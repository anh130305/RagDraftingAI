PROJECT CONTEXT — RAG Soạn thảo Văn bản Hành chính Việt Nam
================================================================

## MỤC TIÊU DỰ ÁN
Xây dựng hệ thống RAG (Retrieval-Augmented Generation) để hỗ trợ soạn
thảo văn bản hành chính Việt Nam, gồm 3 chức năng chính:
  1. Truy xuất điều khoản pháp luật liên quan (legal retrieval)
  2. Gợi ý biểu mẫu hành chính phù hợp (form retrieval)
  3. Few-shot retrieval ví dụ văn bản đã soạn sẵn (example retrieval)

Cột     : doc_id, id, ministry, type_normalized, name,
           chapter_id, chapter_name, article, content
Các bước cleaning:
  1. Drop content = null   → loại 90,706 hàng (17.6%)
  2. Drop exact duplicate  → loại 638 hàng (0.2%)
  3. Drop content duplicate → loại 31,765 hàng (7.5%)
  4. Normalize type → 9 nhóm chuẩn:
       THÔNG TƯ (33.8%), QUYẾT ĐỊNH (21.9%), NGHỊ ĐỊNH (18.2%),
       NGHỊ QUYẾT (15.3%), LUẬT (7.9%), PHÁP LỆNH (1.7%),
       KHÁC (1.2%), VĂN BẢN KHÁC (0.0%), KHÔNG RÕ (0.0%)
  5. Normalize text (whitespace, newline, tab)
  6. Tạo doc_id = SHA256(số_hiệu + article + content_hash[:10])
     → 392,079 unique doc_id, 0 duplicate

4. VECTOR DATABASE — ChromaDB (notebook_final.ipynb)
================================================================

Chạy trên: Kaggle Notebook, GPU Tesla T4
Input : dataset/chunks/*.parquet  (upload lên Kaggle dataset "rag-chunks")
Output: chromadb_output.zip  (tải về, giải nén thành thư mục chromadb/)

Embedding model: intfloat/multilingual-e5-large-instruct
  - Dim       : 1024
  - Max seq   : 512 tokens
  - Asymmetric retrieval:
      Index (passage): prefix "passage: {text}"
      Query (search) : "Instruct: {task}\nQuery: {query}"
  - Distance  : cosine  (embeddings đã L2-normalize)

ChromaDB Collections:
  legal_chunks    → 593,430 chunks  (legal_chunks.parquet)
  forms_chunks    → 10 chunks       (forms_chunks.parquet)
  examples_chunks → 150 chunks      (examples_chunks.parquet)

Retrieval instructions (asymmetric):
  legal   : "Instruct: Tim dieu khoan phap luat Viet Nam lien quan.\nQuery: "
  forms   : "Instruct: Tim mau bieu hanh chinh phu hop voi nhu cau soan thao.\nQuery: "
  examples: "Instruct: Tim vi du van ban hanh chinh tuong tu tinh huong sau.\nQuery: "

Cách dùng local sau khi tải về:
  import chromadb
  client   = chromadb.PersistentClient(path="./chromadb")
  col      = client.get_collection("legal_chunks")  # hoặc forms_chunks / examples_chunks
  # Query:
  results  = col.query(query_embeddings=[q_vec.tolist()], n_results=5,
                        include=["documents","metadatas","distances"])

Insert/Update vài chunk ở local:
  col.upsert(ids=[...], embeddings=[...], documents=[...], metadatas=[...])
  # IDs dùng SHA-256[:24] của "doc_id|chunk_index|text[:200]"

8. BƯỚC TIẾP THEO (chưa thực hiện)
================================================================

  □ Xây dựng RAG pipeline:
      - Nhận user query (mô tả nhu cầu soạn thảo)
      - retrieve_legal()   → top-k điều khoản liên quan
      - retrieve_forms()   → top-k biểu mẫu phù hợp
      - retrieve_examples() → top-k ví dụ few-shot
      - Ghép context → prompt LLM → sinh văn bản

  □ Chọn LLM generation model (GPT-4o / Gemini / local LLM)

  □ Xây dựng prompt template cho từng loại văn bản

  □ Evaluation: đánh giá chất lượng retrieval và generation

  □ (Tuỳ chọn) API / UI cho end-user
